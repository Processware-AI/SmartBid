# 나라장터 입찰 분석 AI Agent 플랫폼 설계안

## 1. 서비스 목표

**나라장터 Open API 기반으로 공공입찰 정보를 자동 수집하고, 사용자의 사업영역에 맞는 입찰을 추천하며, RFP 분석·제안 판단·제안서 초안 생성을 지원하는 AI Agent 플랫폼**입니다.

공식적으로는 조달청의 **나라장터 입찰공고정보서비스**가 물품·용역·공사·외자 입찰공고 목록, 상세정보, 기초금액, 면허제한, 참가가능지역, 변경이력 등을 제공합니다. 업무구분별 오퍼레이션을 맞게 사용해야 정상 응답됩니다. ([데이터.go.kr](https://www.data.go.kr/data/15129394/openapi.do?utm_source=chatgpt.com "조달청_나라장터 입찰공고정보서비스"))

```text
[나라장터 Open API]
        ↓
```

## B. 입찰 적합성 판단 Agent

사용자 회사의 사업 프로필과 입찰공고를 비교합니다.

### 회사 프로필 예시

```text
회사명: Processware AI
주요 역량:
- ASPICE / ISO 21434 / SaMD 사이버보안 컨설팅
- SW 프로세스 개선
- AI 기반 PMO/QMO 시스템
- 교육훈련, 인증 대응, 산출물 점검

희망 입찰:
- 정보화전략계획 ISP
- SW 품질관리
- 프로세스 개선
- 사이버보안 컨설팅
- AI 플랫폼 구축
- 공공 SW 품질 진단
```

### 적합성 점수

|평가항목|배점|
|---|--:|
|사업영역 일치도|30|
|보유 역량 일치도|25|
|예산 규모 적합성|15|
|수행기간 적합성|10|
|참가자격 충족 가능성|10|
|경쟁 강도 추정|10|

결과:

```text
입찰 적합도: 82점
판정: 검토 권장
이유:
- SW 품질관리 및 프로세스 개선 키워드와 높게 일치
- 예산 규모 적정
- 단독 수행 가능성 있음
- 단, 실적 요건 확인 필요
```

## D. Bid/No-Bid 의사결정 Agent

입찰 참여 여부를 결정하기 위한 내부 검토표를 자동 생성합니다.

```text
판정 결과: 조건부 참여

참여 이유:
1. 당사 컨설팅 역량과 과업 범위가 일치
2. AI PMO / 프로세스 자동화 레퍼런스로 확장 가능
3. 공공 SW 품질관리 시장 진입에 유리

불참 리스크:
1. 유사 실적 요건 부족 가능성
2. 상주 인력 요구 시 수익성 저하
3. 제안 준비 기간 부족
```

# 11. 결론

초기 설계 방향은 이렇게 잡는 게 가장 현실적입니다.

```text
1단계: 나라장터 API 수집 + 관심 입찰 추천
2단계: RFP/과업지시서 자동 분석
3단계: Bid/No-Bid 의사결정 자동화
4단계: 제안서 초안 생성
5단계: 낙찰/계약 데이터 기반 시장 인텔리전스
```

가장 먼저 만들 MVP는 **“나라장터 입찰공고 자동 수집 + AI 적합도 평가 + Bid/No-Bid 검토표 자동 생성기”**가 좋습니다.
