# 📌 물품입찰 선별·매칭·실행관리 플랫폼 기획안 (PRD)

## 1. 플랫폼 정의

**AI 기반 공공 물품입찰 실행 플랫폼**

> 나라장터 입찰공고를 자동 수집하고  
> 참여 가능한 공고만 선별 → 공급사 확보 → 입찰참여업체 매칭 →  
> 투찰 검토 → 계약 → 납품 → 정산까지  
> **End-to-End로 실행을 관리하는 플랫폼**

---

## 2. 문제 정의 (Pain Point)

### 기존 문제

```text
1. 입찰공고가 너무 많고 필터링 어려움
2. 참여 가능 여부 판단이 경험 의존
3. 공급사 확보가 가장 큰 병목
4. 낙찰 가능성 판단 어려움
5. 입찰 이후 계약/납품 관리가 분리됨
6. 정보 → 실행까지 연결되지 않음
```

---

## 3. 해결 전략

```text
공고 수집 → AI 분석 → 실행 가능 공고만 선별
→ 공급사 확보 자동화
→ 입찰참여업체 매칭
→ 투찰 전략 지원
→ 계약/납품/정산까지 통합 관리
```

---

## 4. 핵심 가치 (Value Proposition)

### 1) 입찰 “선별”

- 의미 없는 공고 제거
    
- 참여 가능한 공고만 제공
    

### 2) 입찰 “실행”

- 공급사 확보 자동화
    
- 참여조건 생성
    

### 3) 입찰 “수익화”

- 낙찰 가능성 기반 의사결정
    
- 마진 중심 투찰 전략
    

### 4) 입찰 “운영”

- 계약 → 납품 → 정산까지 통합

# 7. 핵심 기능

---

## 7.1 공고 수집 시스템

나라장터 Open API 기반 자동 수집

```text
수집 주기: 10분~1시간
```

수집 항목:

```text
공고번호 / 공고명 / 기관 / 예산
납기 / 규격 / 참가자격 / 계약방법
```

---

## 7.2 AI 리스크 분석 (Hermes Agent)

공고문 + 첨부파일 분석

### 분석 항목

```text
- 납기 리스크
- 규격 리스크
- 인증 요구
- 특정 모델 제한 여부
- 납품 조건
- 계약 조건
```

### 출력

```text
리스크 등급
핵심 위험요소
참여 가능성
```

---

## 7.3 공급사 확보 시스템 (핵심 차별화)

```text
공고 생성
→ 공급사 자동 추천
→ 메신저로 요청 발송
→ 응답 수집
→ 조건 정리
→ 공급사 확정
```

---

## 7.4 입찰참여업체 매칭

```text
공급 확보된 공고만 추천
→ 입찰사 참여 의사 확인
→ 자격 검토
→ 참여 확정
```

---

## 7.5 투찰 전략 엔진

```text
입력:
- 공급 단가
- 물류비
- 마진
- 시장 낙찰률

출력:
- 최소 투찰가
- 권장 투찰가
- 낙찰 확률
```

---

## 7.6 프로젝트 실행관리

```text
입찰 → 낙찰 → 계약 → 납품 → 검수 → 정산
```

상태 관리:

```text
신규 → 분석 → 공급확보 → 참여확정 → 투찰 → 낙찰
→ 계약 → 납품 → 검수 → 정산 → 완료
```
