# 물품입찰 선별·매칭·실행관리 플랫폼 보완 기획안

## 1. 플랫폼 정의

**나라장터 물품입찰 공고를 자동 수집하고, 리스크 분석을 통해 참여 가능한 입찰을 선별한 뒤, 공급사와 입찰참여업체를 매칭하고, 계약·납품·정산까지 실행 관리하는 AI 기반 입찰 운영 플랫폼**입니다.

핵심은 단순 “입찰공고 알림 서비스”가 아니라,

> **공고 발굴 → 리스크 분석 → 공급사 확보 → 입찰사 매칭 → 참여조건 확정 → 계약 → 납품 → 정산**

까지 관리하는 **입찰 실행관리 OS**입니다.

# 4. 사용자 역할

## 1) 관리자

플랫폼 운영자입니다.

주요 기능:

```text
- 나라장터 공고 수집 조건 관리
- 공고별 리스크 분석
- 공급사 등록/승인
- 입찰참여업체 등록/승인
- 공급사-입찰사 매칭 승인
- 계약/납품/정산 상태 관리
- 전체 통계 및 수익 분석
```

---

## 2) 공급사

물품을 실제 공급할 수 있는 업체입니다.

주요 기능:

```text
- 공급 가능 품목 등록
- 제조사/브랜드/모델/단가 등록
- 납기 가능 여부 응답
- 인증서/시험성적서/카탈로그 등록
- 공고별 공급 가능 여부 회신
- 납품 및 정산 상태 확인
```

---

## 3) 입찰참여업체

나라장터에 실제 입찰 참여하는 업체입니다.

주요 기능:

```text
- 추천 공고 조회
- 참여 의사 표시
- 자격/면허/실적 정보 등록
- 투찰 검토
- 계약 진행 확인
- 납품/검수/정산 관리
```

# 5. 핵심 기능 보완안

## A. 공고 수집 관리

나라장터 Open API를 통해 물품 입찰공고를 자동 수집합니다.

수집 기준:

```text
- 품명
- 세부품명번호
- 발주기관
- 수요기관
- 지역
- 예산
- 납기
- 입찰방식
- 계약방법
- 참가자격
- 마감일
```

---

## B. 공고 리스크 분석

Hermes Agent가 공고문과 첨부파일을 분석합니다.

분석 항목:

```text
- 납기 리스크
- 규격 리스크
- 인증 요구 리스크
- 특정 브랜드/모델 제한 여부
- 하자보증 조건
- 납품 장소
- 검수 조건
- 지체상금 조건
- 계약이행보증 여부
- 선금/잔금 조건
```

결과 예시:

```text
리스크 등급: 중간

주의사항:
- 납기 20일로 짧음
- KC 인증서 제출 필요
- 지정 모델 동등 이상 제품 가능
- 납품 장소 3곳 분산
```

---

## C. 공급사 확보 기능

공고별로 공급 가능한 업체를 찾고 응답을 받습니다.

```text
공고 등록
→ 공급사 후보 자동 추천
→ Telegram/KakaoTalk으로 문의 발송
→ 공급 가능 여부 응답
→ 단가/납기/조건 등록
→ 관리자 승인
```

공급사 응답 항목:

```text
- 공급 가능 여부
- 공급 단가
- 납기
- 최소 수량
- 인증 가능 여부
- 대체 모델 제안
- 특이 조건
```

---

## D. 입찰참여업체 매칭

공급사가 확보된 공고를 입찰참여업체에게 추천합니다.

매칭 기준:

```text
- 입찰 자격
- 업종/품목 적합성
- 지역 제한
- 실적 요건
- 보증/계약 가능성
- 과거 참여 이력
- 신뢰도 점수
```

결과:

```text
참여 가능 공고 생성
→ 입찰참여업체 알림
→ 참여 의사 확인
→ 관리자 검토
→ 참여 확정
```

---

## E. 투찰 검토 지원

물품입찰은 가격·마진·납기 리스크가 중요합니다.

검토 항목:

```text
예정가격
공급단가
물류비
플랫폼 수수료
입찰참여업체 마진
예상 투찰가
최소 수익률
낙찰 가능성
```

계산 구조:

```text
공급가 + 물류비 + 제비용 + 업체마진 + 플랫폼 수수료 = 최소 투찰 가능가
```

# 6. Hermes Agent 역할

Hermes Agent는 플랫폼의 “AI 실무자” 역할을 맡습니다.

## Agent 구성

```text
1. Notice Collector Agent
- 나라장터 공고 수집
- 신규/변경 공고 감지

2. Risk Analysis Agent
- 공고문/규격서/첨부파일 분석
- 리스크 요약

3. Supplier Matching Agent
- 공급 가능 업체 추천
- 공급사 응답 정리

4. Bidder Matching Agent
- 입찰참여업체 추천
- 참여 가능성 판단

5. Pricing Agent
- 공급가, 마진, 수수료 기반 투찰가 검토

6. Execution Manager Agent
- 일정, 서류, 계약, 납품, 정산 상태 추적

7. Messenger Agent
- Telegram/KakaoTalk 알림 발송
- 응답 수집
- 미응답자 재알림
```

# 13. 최종 컨셉

이 플랫폼의 본질은 다음입니다.

> **“입찰 정보를 알려주는 서비스”가 아니라, 공급사와 입찰참여업체를 연결하고 실제 계약·납품·정산까지 관리하는 물품입찰 실행 플랫폼”**

초기에는 **관리자 중심의 반자동 운영 플랫폼**으로 시작하고, 이후 Hermes Agent를 고도화해 **입찰 실행 Copilot**으로 확장하는 방향이 가장 현실적입니다.
