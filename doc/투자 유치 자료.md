# 📊 1. 투자자용 피치덱 (10장 구조)

## 1. Title

**AI 기반 공공입찰 실행 플랫폼**

> “입찰 정보를 넘어, 실제 수주까지 책임지는 AI”

- 물품입찰 선별·매칭·실행관리 플랫폼
    
- B2B SaaS + Transaction 모델
    

---

## 2. Problem

현재 조달 시장의 구조적 문제

```text
- 공고는 넘치지만 ‘돈 되는 입찰’은 찾기 어려움
- 공급사 확보가 가장 큰 병목
- 낙찰 가능성 판단이 경험 의존
- 입찰 이후 실행(계약/납품/정산)이 분리됨
```

👉 핵심:  
**“정보는 있지만 실행이 없다”**

---

## 3. Solution

```text
공고 수집
→ AI 리스크 분석
→ 공급사 확보
→ 입찰사 매칭
→ 투찰 전략
→ 계약/납품/정산 관리
```

👉 한 줄

> “입찰 → 수주 → 실행까지 자동화”

---

## 4. Product

플랫폼 핵심 기능

```text
- AI 공고 분석
- 공급사 자동 확보
- 입찰사 매칭
- 투찰 전략 추천
- 실행관리 (계약~정산)
```

👉 차별 포인트

> “입찰 실행까지 연결하는 유일한 플랫폼”

---

## 5. Market

## TAM / SAM / SOM

### TAM (전체 조달시장)

- 국내 공공조달 규모: 약 200~300조원
    

---

### SAM (물품 조달 시장)

- 약 100조원
    

---

### SOM (초기 타겟)

- 중소 물품 입찰 시장
    
- 약 5~10조원
    

---

👉 초기 목표

```text
SOM의 1% = 500억 규모
```

---


# 📊 2. 시장 분석

## 1. 시장 구조

```text
공공기관 → 입찰공고 → 입찰참여업체 → 공급사 → 납품
```

문제:

👉 이 과정이 **분절됨**

---

## 2. 현재 시장 플레이어

### ① 입찰정보 서비스

- 나라장터 (공식)
    
- 민간 입찰정보 사이트
    

👉 역할

```text
정보 제공
```

---

### ② 입찰 대행업체

👉 역할

```text
입찰 대신 수행
```

문제:

```text
- 사람 중심
- 확장성 없음
```

---

### ③ 공급사/유통사

👉 역할

```text
제품 공급
```

문제:

```text
- 입찰 접근성 낮음
- 수요 연결 부족
```

---

## 3. 기회

👉 현재 시장의 공백

```text
“정보 → 실행” 연결 플랫폼 없음
```

---

👉 그래서 생기는 기회

```text
입찰 실행 플랫폼
= 완전히 새로운 카테고리
```

# 📊 3. 경쟁사 비교

## 경쟁 구도

|구분|기존 서비스|우리 플랫폼|
|---|---|---|
|공고 제공|O|O|
|공고 분석|X|O|
|공급사 연결|X|O|
|입찰사 매칭|X|O|
|투찰 전략|X|O|
|실행관리|X|O|

---

## 주요 경쟁군

### 1. 나라장터

역할:

```text
공고 게시
```

한계:

```text
- 분석 없음
- 실행 없음
```

---

### 2. 민간 입찰정보 사이트

역할:

```text
입찰 정보 제공
```

한계:

```text
- 단순 검색
- 실행 연결 없음
```

---

### 3. 입찰 컨설팅

역할:

```text
사람이 대신 수행
```

한계:

```text
- 비확장성
- 고비용
```

---

## 차별화 포지셔닝

```text
기존:
입찰정보 플랫폼

우리:
입찰 실행 플랫폼
```

---

## 경쟁 우위 (Moat)

### 1. 데이터 축적

```text
공고 + 공급 + 입찰 + 가격 + 낙찰 데이터
```

---

### 2. 네트워크 효과

```text
공급사 ↑ → 입찰 성공 ↑
입찰사 ↑ → 공급사 수익 ↑
```

---

### 3. Workflow Lock-in

```text
계약/납품/정산까지 관리
→ 이탈 어려움
```
